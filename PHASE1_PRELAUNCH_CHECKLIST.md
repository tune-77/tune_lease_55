if ENABLE_OBSIDIAN_CACHE:
    # キャッシング使用
    obsidian_hits = cached_collect_obsidian_context(...)

    # キャッシング無効化（フォールバック）

### 2️⃣ エラーハンドリング

**問題**: キャッシングでエラーが発生した場合、AI チャットが止まるか？

**確認項目**:
```
❓ キャッシュエラー時に自動フォールバックするか？
❓ エラーログは記録されるか？
❓ ユーザーには見えないか？
```

**推奨**:
```python
def build_chat_reply(...):
    try:
        if ENABLE_OBSIDIAN_CACHE:
            obsidian_hits = cached_collect_obsidian_context(...)
        else:
            obsidian_hits = collect_obsidian_context(message)
    except Exception as e:
        logger.error(f"CACHE_ERROR: {e}")
        # フォールバック
        obsidian_hits = collect_obsidian_context(message)  # キャッシュなし
        logger.info("Fallback to non-cached search")
```

**期待**: ユーザーには影響なし（遅くなるだけ）

### 5️⃣ キャッシュサイズ管理

**問題**: キャッシュがメモリを食い尽くさないか？

**確認項目**:
```
❓ 最大キャッシュサイズの制限はあるか？
❓ キャッシュサイズを監視しているか？
❓ メモリ上限に達した時の動作は？
```

**推奨**:
```python
# obsidian_context_cache.py に最大サイズ制限を追加
class ObsidianContextCache:
    def __init__(self, ttl_seconds: int = 300, max_size: int = 1000):
        self.cache = {}
        self.ttl = ttl_seconds
        self.max_size = max_size  # 最大 1000 エントリ
    
    def set(self, query: str, data: dict) -> None:
        if len(self.cache) >= self.max_size:
            # 最も古いエントリを削除（LRU）
            oldest_key = min(
                self.cache.keys(),
                key=lambda k: self.cache[k]["created"]
            )
            del self.cache[oldest_key]
            logger.warning(f"Cache eviction: max size reached")
        
        # 新しいエントリを追加
        self.cache[key] = {...}
```

## ⚠️ 「これだけは避けるべき」リスト

```
❌ 本番環境で全ユーザーに一度に展開
   → 問題が発生したら対応できない
   → ローリング展開を厳守

❌ ログに質問内容や Vault 内容を記録
   → プライバシー漏洩のリスク
   → メタデータのみ記録（長さ、件数等）

❌ キャッシュサイズの上限がない
   → メモリ枯渇
   → システムクラッシュ
   → 上限を 1000 エントリに設定

❌ エラーハンドリングなしで展開
   → キャッシュエラーで AI チャット停止
   → フォールバック必須

❌ ロールバック手順なしで展開
   → 問題が起きても戻せない
   → 環境変数フラグで即座に無効化可能に
```
