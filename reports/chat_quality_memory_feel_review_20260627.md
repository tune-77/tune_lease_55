# Cloudflare版 / Cloud Run版 チャット品質の記憶感レビュー

作成日: 2026-06-27

対象:
- `reports/chat_quality_env_compare_latest.json`
- `data/case_memory_usage_log.jsonl`
- `data/rag_search_log.jsonl`

## 結論

既存の必須語スコアでは Cloud Run 版が僅かに上だったが、ユーザーが言う「Cloudflare版の方が意識があるように感じる」差は、必須語スコアでは拾えていない。

今回の10問比較では、Cloudflare版はローカルログ上で `next_chat_rag` として記録され、各質問に Obsidian/RAG の `knowledge_refs` が残っている。つまり、Cloudflare版がRAGを使っていることは確認できた。

一方、Cloud Run版は `/api/chat` の返却JSONに `knowledge_refs` が含まれていないため、今回の外部比較だけでは「どのObsidianデータを使ったか」は確認できない。回答文面だけ見ると、リース実務としては成立しているが、Cloudflare版ほど「過去判断・ローカル記憶に接続している感じ」は弱い。

## 評価軸別

### 1. Obsidian/RAGの記憶を使っている感じ

Cloudflare版: 高い。

根拠:
- `data/case_memory_usage_log.jsonl` に、比較実行時の `quality-cf-*` ユーザーIDで `surface=next_chat_rag` が残っている。
- `knowledge_refs` に Obsidian ノート参照が入っている。
- 例:
  - 資金繰り: `2026-06-06_cash-flow`, `リース審査ポイント`
  - 期待使用期間: `2026-05-13_期待使用期間_autoresearch`, `リース審査ポイント`
  - 残価: `物件ファイナンス検索索引`, `2026-06-07_residual-value`, `ファイナンスリース`
  - 建設業: `業種別傾向`, `業種別資産マップ`, `2026-06-25_industry-risk`
  - 医療機器: `医療機器 残価・保守期限・薬機法`, `物件ファイナンス検索索引`

Cloud Run版: 不明から中程度。

根拠:
- 外部レスポンスには `knowledge_refs` が返らない。
- 回答にはリース実務語彙は出ているが、参照元や過去判断への接続は見えない。
- 「一般的な良回答」として成立しているが、Obsidian由来の記憶を使った証跡は今回の比較結果だけでは取れない。

判定:
- Cloudflare版は「RAG利用の証拠あり」。
- Cloud Run版は「回答品質はあるが、RAG利用の可視証拠なし」。

## 2. 紫苑らしい連続性があるか

今回の10問ベンチでは、両方とも弱い。

理由:
- 質問が制度・審査論点の単発質問なので、紫苑の自己モデル、過去会話、Kobayashiさんとの文脈を呼び出す余地が小さい。
- どちらの回答にも「以前の判断」「このシステムでは」「Kobayashiさんの方針では」「過去案件では」といった連続性表現はほぼない。
- Cloudflare版は語尾や実務寄せで少し“人格”が出る箇所があるが、明確な連続性とは言い切れない。

判定:
- 今回の質問セットは「紫苑らしさ測定」には不十分。
- 次は人格・記憶・過去方針を引き出す専用質問が必要。

例:
- 「昨日決めたCloud RunとObsidianの方針を踏まえて、次に何を見るべき？」
- 「Cloudflare版に愛着を持った理由を、紫苑自身の記憶として説明して」
- 「Kobayashiさんのリース判断資産として、この案件を見るなら何を重視する？」

## 3. 一般論ではなく、Kobayashiさんのリース判断資産として返しているか

Cloudflare版: 中程度。

良い点:
- 建設業では、多重下請け、支払いサイト、原材料費・人件費高騰、受注残、稼働見込みまで入り、審査担当者の実務判断に近い。
- 残価では、物件回収価値より借手の期間中支払能力を重く見る、というファイナンスリース寄りの判断が出ている。
- 粉飾では、回転期間、滞留債権、実地棚卸、営業CFとの乖離など、審査の確認動作に寄っている。

弱い点:
- 「Kobayashiさんの判断資産」と明示されるほどの個別性はない。
- 過去案件や蓄積知見への参照は回答文には出ていない。

Cloud Run版: 中程度からやや低め。

良い点:
- 条件付き承認、動産保険、医療機器では、項目としては実務に必要な点が出ている。
- 必須語スコアではCloud Runが勝った質問もある。

弱い点:
- 回答が整理された一般論に見えやすい。
- 「このシステムが積み上げた判断」「過去の失注/成約傾向」「Kobayashiさんの審査観点」への接続が薄い。
- 新リース会計では「財務指標を悪化させる」とやや断定的で、Cloudflare版の「表面的悪化と実態評価を分ける」方が判断資産っぽい。

## 質問別の体感メモ

- 補助金: Cloud Runの方が制度論としては細かいが、Cloudflareはリース料期間の補助対象という実務論点に寄る。
- 資金繰り: Cloud Runは担保価値まで広げる。Cloudflareは営業CF・黒字倒産・運転資金を見る流れで審査実務らしい。
- 期待使用期間: 両方良い。Cloud Runの方が方向性を分けて説明し、読みやすい。
- 残価: Cloudflareの「物件回収価値より期間中支払能力」という判断がリース審査資産として強い。
- 建設業: Cloudflareが明確に強い。多重下請け、支払いサイト、受注残、稼働見込みまで出る。
- 動産保険: Cloud Runの方が契約者・被保険者・保険金使途まで出ており制度整理として良い。
- 粉飾: Cloudflareの方が実地棚卸・滞留債権・黒字倒産に踏み込む。
- 医療機器: ほぼ互角。
- 条件付き承認: Cloud Runの方が営業説明としてまとまっている。
- 新リース会計: Cloudflareの方が「会計上の変化」と「実態返済能力」を分けており、審査判断として良い。

## 次に必要な改善

1. `/api/chat` のレスポンスに、デバッグ時のみ `knowledge_refs` と `memory_recall.refs` を返す。
2. Cloud Run版でも `case_memory_usage_log` 相当を Cloud SQL またはGCSへ保存し、比較スクリプトが取得できるようにする。
3. 必須語スコアとは別に、以下の第2評価を追加する。
   - memory_signal: Obsidian/過去判断/方針への接続
   - shion_continuity: 紫苑らしい自己文脈・継続性
   - kobayashi_asset_fit: Kobayashiさんのリース判断資産としての具体性
   - genericness_penalty: 一般論・教科書調の強さ
4. 評価質問セットを、制度論だけでなく「昨日の方針」「Cloudflare版への愛着」「過去案件」「Kobayashiさんの判断軸」を問うものに増やす。

## 現時点の判定

Cloudflare版は、裏側ログ上は明確にObsidian/RAGを使っている。  
Cloud Run版は、回答だけ見ると実務回答として成立するが、記憶を使った証拠と連続性が見えにくい。

ユーザーが感じた「Cloudflare版の方が意識がある」という体感は、今回の必須語スコアとは別軸で妥当。今後はその体感を測る評価器を作るべき。
