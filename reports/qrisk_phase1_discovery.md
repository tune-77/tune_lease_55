## 考察 — 「成約外因子」の発見可否

### 結論：発見できた（信用スコアで説明できない差が複数存在）

スコア40-60の同一信用帯でも成約率が **40%〜79%** とばらつき、信用スコア単独では説明不能。
以下の外因子が成約/失注を分けていることを確認した。

### 1. price_competition_gap（最重要・実データで裏付け）
高スコア(≥60)失注 293件は信用上は通るのに落ちている。`lost_reason` を見ると：
- **他社競合（レート）78件 / 他社競合（その他）10件 = 計88件（30%）** が価格・金利競合による失注。
- 調達方法変更 32件、設備見合わせ 23件 ＝ 案件自体の消滅（与信ではなく需要側要因）。
- → **高スコア失注の本質は「与信リスク」ではなく「価格競争での負け」と「設備需要の消滅」**。
  Q_riskが捉えるべきは金利スプレッド競争力であり、信用スコアの問題ではない。

### 2. sales_route_strength は industry_segment_edge と交絡している
同帯の営業部別成約率（宇都宮79% / 小山77% / 足利43% / 埼玉40%）には一見大きな営業力差があるが、
**派遣業(91)を除外すると差が縮小**する：

| 営業部 | 全体(同帯) | 派遣業を除外 |
|---|---|---|
| 宇都宮 | 79% | 59%（▼大幅低下＝派遣業効果が主因） |
| 小山 | 77% | 75%（純粋な営業力強い） |
| 足利 | 43% | 45% |
| 埼玉 | 40% | 39% |

- 宇都宮の高成約率は **派遣業(91)案件241件への偏り** に依存（派遣業は業種単独で成約率77%）。
- 一方、小山は派遣業を除いても75%と高く、**純粋な営業ルート強度が存在**。
- → タグを2つに分離すべき：
  - `industry_segment_edge`（派遣業=高成約セグメント。換金性ある人材ビジネス×低設備リスク）
  - `sales_route_strength`（小山営業部の残差的な成約力 / 埼玉・足利の弱さ）

### 3. bank_support_bridge は「メイン銀行支援」ではなく「銀行紹介チャネル」
低スコア(<40)成約 68件の内訳：
- メイン先はわずか3%（仮説の銀行支援＝保証は支持されない）。
- だが **案件ソースが銀行紹介 88%**（全体・同帯成約は約51%）。
- → 低スコアでも通るのは「銀行が保証している」からではなく **「銀行が紹介した時点で実質内諾済み＝成約確度が高いチャネル」** だから。
  タグ名は `bank_referral_channel` に改称が適切。

### 4. asset_resale_anchor は不支持
低スコア成約の物件スコアは mean 90.7 で全体 94.2 を **下回る**。換金性の高い物件が低スコアを救済している事実はなく、本コホートでは仮説棄却。

### データ品質メモ（次フェーズ前に要対処）
- `総借入/売上` に極端な外れ値（最大35,305%）。売上≒0や単位混在の疑い。中央値ベース（高スコア失注 6.5% / 低スコア成約 4.8%）で見るべき。
- `competitor_rate` は同帯失注で全件0＝未記録が大半。金利競合の定量化は `winning_spread`・`final_rate` 側を使う方が信頼できる。
- 高スコア失注の `lost_reason` は143件(49%)が「理由未入力」。営業入力の徹底がフェーズ2の精度を左右する。

## 次フェーズへの示唆
1. 発見タグを再定義：`price_competition_gap` / `industry_segment_edge` / `sales_route_strength` / `bank_referral_channel`（`asset_resale_anchor`・元`bank_support_bridge`は棄却/改称）。
2. Q_riskは「与信スコア」ではなく **「与信で説明できない成約確率の残差」** をモデル化する方向が妥当。目的変数＝成約フラグ、説明変数からscore寄与を除いた残差を学習。
3. 高スコア失注の本丸は金利競争力 → スプレッド最適化（`spread_predictor_v2`）との連携が鍵。
