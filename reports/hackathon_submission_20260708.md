# ハッカソン提出案: Tune式 リース審査アシスタント

## タイトル

審査担当者の判断力を育てるAI - Tune式 リース審査アシスタント

## 概要

Tune式 リース審査アシスタントは、リース案件の入力情報から信用スコア、リスク論点、承認条件、稟議コメント案までを一気通貫で生成する審査支援AIです。

単に「承認・否決」を自動判定するのではなく、審査担当者が何を確認し、どの条件なら前に進められるかを整理します。AI人格「紫苑」が、過去案件やレビュー履歴を踏まえて、案件ごとの見落としや反面教師を提示します。

## 解決したい課題

リース審査では、財務数値だけでなく、物件の回収可能性、銀行支援、競合条件、営業経緯、過去類似案件との比較など、多くの論点を短時間で整理する必要があります。

しかし実務では、担当者の経験差によって稟議品質がばらつきます。スコアは出せても、「なぜ危ないのか」「どうすれば通せるのか」「審査部に何を聞かれるのか」まで一貫して説明するのは難しいのが現状です。

このアプリは、審査担当者の思考プロセスをAIで補助し、判断の質と説明力を底上げすることを目指しています。

## 主な機能

- 案件情報、財務情報、定性情報、物件情報の入力
- リース審査スコアと判定の生成
- 財務・業種・e-Stat・物件リスクを統合した分析
- 紫苑レビューによる案件コメント
- 過去案件・反面教師の会社名ハイライト
- 案件作戦盤による承認条件、確認質問、稟議戦略の提示
- 稟議コメント案の生成
- 通常AIチャットとリース知性体の記憶共有

## 新規性

一般的な審査AIは、スコアやリスク判定を出すことに寄りがちです。

本アプリは、スコアそのものよりも「審査担当者がどう考えるべきか」に重点を置いています。紫苑レビューは過去の案件や失敗パターンを踏まえ、案件作戦盤は実際に次に取るべき行動へ落とし込みます。

つまり、AIが判断を奪うのではなく、人間の判断力を鍛える設計です。

## 技術構成

- Frontend: Next.js / React / TypeScript
- Backend: FastAPI / Python
- AI: Gemini API 連携
- Data: リース審査ロジック、過去案件、業種ベンチマーク、e-Stat文脈
- Infrastructure: Google Cloud Run / Artifact Registry
- Memory: Obsidian連携、AIチャット記憶、紫苑レビュー履歴

## デモ手順

1. 審査分析画面を開く
2. デモ案件を読み込む
3. 審査エンジンを実行する
4. スコアと判定を確認する
5. 紫苑レビューを生成する
6. 過去案件の会社名ハイライトを見る
7. 案件作戦盤で、承認条件・質問・稟議戦略を見る
8. 稟議コメント案に落とし込めることを示す

## 3分デモ台本

### 0:00 - 0:25 課題提示

リース審査では、単にスコアが高いか低いかだけでは判断できません。

物件の回収可能性、銀行支援、競合条件、営業経緯、過去案件との比較などを短時間で整理し、稟議として説明できる形にする必要があります。

そこで作ったのが、審査担当者の判断力を育てるAI、Tune式 リース審査アシスタントです。

### 0:25 - 0:55 案件入力

まず、営業担当が案件情報を入力します。

企業情報、財務数値、物件情報、取引背景を入れると、AIが審査に必要な論点として整理します。

今日はデモ案件を読み込んで、そのまま審査を実行します。

### 0:55 - 1:30 審査結果

審査エンジンを実行すると、スコアと判定が出ます。

ここでは単純な点数だけではなく、営業利益率、自己資本比率、業種平均との差、リース負担、景気の向きなどを統合して見ています。

重要なのは、AIが「この案件は危ない」と言うだけではなく、どの論点を追加確認すべきかまで分解する点です。

### 1:30 - 2:10 紫苑レビュー

次に紫苑レビューを生成します。

紫苑は、通常のAIチャットや過去レビューの記憶を参照しながら、この案件をどう見るべきかコメントします。

過去案件の会社名はレビュー内で色を変えて強調されるため、「どの経験を根拠に見ているのか」がわかります。

AIがブラックボックスで答えるのではなく、担当者が過去の判断とつなげて理解できるようにしています。

### 2:10 - 2:45 案件作戦盤

右側の案件作戦盤では、紫苑レビューを補助する形で、今日やるべき3手、審査部からのツッコミ予測、顧客に聞くべき質問、稟議で使える一言を出します。

ここがこのアプリの一番の特徴です。

AIが承認可否を決めるのではなく、人間が判断するための材料と行動を整理します。

### 2:45 - 3:00 まとめ

このアプリは、審査AIではなく、審査担当者の思考を育てるAIです。

スコア、過去案件、紫苑レビュー、案件作戦盤をつなげることで、経験の浅い担当者でも、質の高い稟議と判断に近づけます。

## 審査員向けの強調ポイント

- スコアリングだけで終わらず、承認条件と稟議文言まで落とし込む
- 過去案件・反面教師をレビュー内で可視化する
- AI人格「紫苑」が通常チャットと審査分析をまたいで記憶を共有する
- 人間の最終判断を前提にした、実務寄りのAI設計

- 稟議承認後の結果をフィードバックし、レビュー品質を継続改善する
- 金融機関・リース会社向けの審査教育ツールとして展開する
