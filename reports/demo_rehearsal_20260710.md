## 発見した問題（優先度順・未修正）

### P1: デモ画面に直接見える

1. **承認バッジとめぶきポップアップの矛盾**（デモ冒頭のメイン案件で毎回発生）
   - デモ精密工業で緑の「承認」バッジと同時に、めぶきが「スコア 79.4 点。少し工夫が必要です。」と表示。
   - 原因: `frontend/src/app/screening/page.tsx` の `handleSubmit` 内で `score_base` を **80/50 のハードコード閾値**で分岐。
     CLAUDE.md の規約どおり `scoring_core.APPROVAL_LINE`（71点）系に揃っていない（71の二重定義問題の再発形）。
2. **「境界」枠のデモ北関東物流が承認圏内（83.4点）になる**
   - デモUIの売り文句「3件で通る・境界・慎重の違いを見てください」が崩れる。
   - 本環境ではデモ精密工業も 81.9 点（7/1 の Work Log では 89.9 点）でスコアに環境差がある。
     **本番機で3案件のスコアを実測し、境界案件が60点台に落ちるか確認が必要**。落ちないなら案件データ側の調整を検討。
3. **ページ読み込みのたびに `/api/screening-experience-cases` が3回 422**
   - 原因: フロントが結果未取得時に `score=""`（空文字）を送り、FastAPI の `Optional[float]` が拒否。
   - UI はハードコードのフォールバック表示で隠れるが、コンソールエラーになり、dev 起動だと Next の赤バッジ（N 1 Issue）が画面に出る。
   - 修正は1行級（score が空なら params から外す、またはサーバー側で空文字許容）。

### P2: 障害時の見え方（デモ中の保険）

4. **Gemini キー未設定/障害時、紫苑レビューは赤文字の汎用エラーのみ**
   - `/api/chat`（screening_review 経路）は 500「内部エラーが発生しました」を返す。
     `api/main.py` に既にある「APIキー未設定」丁寧文言（503系）のハンドリングがこの経路では効いていない。
   - デモ中にキー/クォータが死んだ場合、原因切り分けができない。文言の出し分け改善余地。
5. **`ReportGenerator.tsx` が生 axios を使用**（`apiClient` 不使用、`API_BASE` は import して未使用）
   - 同一オリジン構成では動くが、`NEXT_PUBLIC_FASTAPI_BASE_URL` 分離構成でデモする場合は稟議書生成だけ落ちる。

### P3: 環境再現性（当日マシン変更時の罠）

6. **`requirements.txt` に `slowapi` が無い**（`pyproject.toml` が正）
   - フレッシュ環境で `pip install -r requirements.txt` だと `api/main.py` が ImportError で起動しない。
7. **`next dev` が Next.js 16 でフラグ明示必要**
   - `package.json` の dev スクリプトは素の `next dev` のままで、turbopack 設定エラーで起動失敗。`next dev --webpack` なら起動する。
   - 本番は standalone ビルド（`--webpack` 指定済み）なので影響なし。当日 dev 起動する場合のみ注意。
