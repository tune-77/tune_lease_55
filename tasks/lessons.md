## 運用ルール（2026-03-18 設定）

### 基本の動き方
- 手順が3ステップ以上 or 重要な判断が必要 → 必ず plan モードで計画してから動く
- 何かおかしくなったら → すぐ STOP して再計画（押し切らない）
- 完了扱いにするのは「動くことを証明してから」

### 説明の仕方
- 相手は技術者ではない前提で話す
- 技術用語を使う場合は、必ず言い換えを先に書く
- バックグラウンドで動く処理は「何をしているか」を短く要約して伝える
- エラーが出たら必ず報告し、「どう直せばいいか」を分かる言葉で案内する

### ファイル・コマンドの安全ルール
- **既存ファイルを編集する前に必ず確認**：「〇〇を上書きしますが、よろしいですか？」
- **削除コマンド（rm / del / rmdir）は原則禁止**：必要な場合は理由を説明して承認を得る
- **パッケージ追加（pip install など）は事前説明と承認が必要**
  - 何をインストールするか・なぜ必要か・影響範囲を説明する
- **専門的なコマンドは実行前に日本語で説明してから確認を取る**

### タスク管理
- 作業開始前に `tasks/todo.md` に手順リストを書く
- 進めながら完了項目を更新する
- 修正が入ったら `tasks/lessons.md`（このファイル）に記録する

### コードの品質
- シンプル最優先：最小限の手数で最大効果
- 必要な箇所だけ触る（余計な変更を持ち込まない）
- 「もっとスッキリした方法はあるか？」と毎回問う
