# Hackathon Submission Copy

## 1-line

紫苑は、リース審査で人間が残した判断コメントを、次の案件で再利用できる「判断資産」に変えるAIです。

## 3-line

普通の審査AIは、案件ごとにスコアやコメントを出して終わります。
紫苑は、人間が「なぜ迷ったか」「何を確認したか」「どんな条件なら通せるか」を保存します。
その判断を、次の案件の確認点・承認条件・稟議コメントとして再利用します。

## Short Description

紫苑（SHION）は、リース審査の判断を使い捨てにしないAIエージェントです。案件ごとのスコアリングだけでなく、人間の違和感、修正、承認条件、結果を検疫し、再利用可能な判断資産へ変換します。蓄積された判断資産は、次の案件で確認質問、承認条件、反証、稟議文面として組み直されます。

## Judge Hook

この作品の評価ポイントは、モデル単体の賢さではありません。AIの回答、人間の判断、結果データを観測し、検疫し、次回判断へ戻す Human-in-the-loop DevOps です。リース審査を実証フィールドに、業務AIが使われながら判断品質を改善する仕組みを実装しました。

## Video Opening

このAIは、審査判断を一回で終わらせません。

人間が残した迷い、確認、修正、承認条件を保存し、次の案件で使える判断資産に変えます。

それが、リース知性体「紫苑」です。

## What To Emphasize

- 人間の審査コメントを、AIが再利用できる判断資産に変える
- 次の案件で、確認点・承認条件・稟議コメントとして戻す
- 危ない学習は、検疫DBと人間レビューを通してから本番へ昇格する
- スコアを出すだけでなく、判断の根拠と改善履歴を残す
