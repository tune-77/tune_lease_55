Implementation Plan

リース審査AI 期間・保険チェック実装計画

法定耐用年数、期待使用期間、リース期間の3点チェックと、動産保険/再リース保険対象外警告を、 既存スコアリングに安全に追加するための設計計画。初期段階ではスコアを直接変更せず、審査結果の警告・根拠表示・ログ保存に限定する。

対象: tune_lease_55 作成日: 2026-05-13 状態: 計画のみ・未実装

1. 方針

最初は減点しない

既存の借手スコア、物件スコア、成約確率ロジックと衝突させない。まずは警告表示とログ保存に限定する。

既存資産を使う

expected_usage_period.py にある期待使用期間・法定耐用年数ロジックを中核にする。

あとでスコア反映

運用実績を見てから、物件スコア減点、残価評価補正、審査ステータス強制変更へ段階的に進める。

2. 対象機能

機能 見る項目 初期動作 将来拡張
3点チェック legal_useful_life, expected_use_period, lease_term OK/注意/要確認/高リスクを表示 物件スコア補正、残価評価補正
動産保険警告 保険対象可否、対象外理由、物件カテゴリ 対象外・不明時に警告 契約条件の必須確認、保険入力の必須化
再リース保険警告 再リース予定、満了後継続使用、再リース回数 再リース時に保険対象外可能性を表示 再リース料・残価・回収可能性評価へ接続
初期実装では、審査結果の説明品質と事故防止を優先する。スコアに反映すると否決・承認境界が変わるため、後続フェーズで校正する。

3. 既存コードへの接続点

ファイル 役割 変更方針
expected_usage_period.py 期待使用期間、法定耐用年数、リース期間適合度の計算 既存関数を利用。必要なら結果形式だけ拡張。
components/form_apply.py 物件、契約期間、契約条件の入力 保険対象、再リース予定、対象外理由の入力欄を追加。
components/score_calculation.py 審査実行、結果辞書生成、ログ保存 lease_rule_checks を結果に追加。スコアは変更しない。
components/analysis_results.py または結果表示部 審査結果画面の表示 警告カードとして表示。出典と確認事項を明示。
data_cases.py 案件ログ保存 既存の inputs / result に格納できるならDB変更なしで開始。

4. 入力項目案

項目 型 初期値 用途
is_re_lease boolean false 再リース保険対象外警告、再リース余地評価
insurance_applicable category unknown 動産保険対象/対象外/不明の判定
insurance_exclusion_reason category none 車両、再リース、特殊物件、屋外設置、その他を記録
usage_period_item_name string asset_nameから推定 期待使用期間マスタとの照合
lease_term number 既存入力 3点チェックの中心項目

6. 判定ルール案

判定 条件 表示文言
高リスク リース期間が法定下限未満 税務・会計上の取扱確認が必要です。
注意 リース期間が法定下限に近い 期間設定の妥当性を確認してください。
注意 リース期間が期待使用期間を超える 故障、陳腐化、残価低下の可能性があります。
要確認 再リース予定かつ保険対象不明 再リース中の動産保険対象可否を確認してください。
OK 法定下限以上、期待使用期間内、保険対象確認済 期間・保険面の重大な警告はありません。

7. 画面表示案

  1. 審査結果画面のスコア下に「期間・保険チェック」セクションを追加する。
  2. 重大度順に、赤、黄、緑のカードで表示する。
  3. 各カードには、判定、理由、確認すべきこと、根拠を表示する。
  4. 警告はAIコメント生成にも渡し、審査所見へ自然文で反映する。
表示例: 「本件はリース期間60ヶ月に対し法定耐用年数5年で、期間後半の残価・再リース余地は限定的です。再リースを前提にする場合は保険対象可否を確認してください。」

8. 実装フェーズ

Phase 内容 完了条件
Phase 1 チェック関数と結果データ構造を追加。スコア変更なし。 審査結果に lease_rule_checks が出る。
Phase 2 入力画面に保険/再リース項目を追加。 保険警告が入力値に応じて変わる。
Phase 3 結果画面に警告カードを追加。 審査官が確認事項を一目で把握できる。
Phase 4 AIコメント生成へ連携。 所見に期間・保険リスクが反映される。
Phase 5 実績を見てスコア補正を検討。 誤警告率、審査判断への影響を確認済み。

9. テスト観点

  • 期待使用期間マスタに存在する物件で、期間チェックが出る。
  • 期待使用期間マスタに存在しない物件で、エラーではなく「未判定」になる。
  • リース期間0、未入力、120ヶ月などの境界値で落ちない。
  • 再リース予定かつ保険対象不明の場合に警告が出る。
  • 保険対象外理由が車両、屋外、特殊物件、再リースで表示文言が変わる。
  • スコア値はPhase 1では変わらない。
  • 案件ログに lease_rule_checks が保存される。

10. 次の判断

実装に進む場合は、最初に lease_rule_checks.py のような小さな独立モジュールを作り、 expected_usage_period.py を呼び出してチェック結果を返す構成がよい。 UI、スコア計算、ログ保存を直接絡める前に、純粋関数としてテストできる形にする。
