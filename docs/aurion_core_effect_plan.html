Planning Document

AURION CORE

効果可能性検討・段階導入計画 / AI Collaborative Evolution Protocol v2.1

対象: リース審査AI 方針: 実装前計画 作成日: 2026-05-13
結論 機能分解 効果可能性 ロードマップ 優先順位 推奨構成
結論: AURION CORE は審査AIの「心臓」ではなく、まず「感覚器官」として導入する。 既存の RF/LR 主モデル、ベイズ推論、定性評価、物件スコアを置き換えるのではなく、Q_risk・競合圧力・ステルス競合・市場の歪みを観測し、人間が判断できる言葉へ翻訳する上位レイヤーとして計画する。

1. 機能分解

実装可能性: 高

Q_risk: 財務矛盾の位相検知

既存の量子インスパイア解析を活かし、財務ペア間の干渉乖離、外挿フラグ、残差シグナルを「高スコア案件の違和感」として提示する。

  • 主判定モデルではなく二次審査・説明レイヤーに限定する。
  • 特異点は「高寄与ペア」「外挿変数」「残差ピーク」と実務定義する。
  • 高スコアかつ高 Q_risk の案件を優先表示する。
実装可能性: 中〜高

LV_Environment: 競合環境モデル

ロトカ・ヴォルテラ方程式をいきなり本番予測に使わず、まずは業種別・営業部別の競合圧力指数として導入する。

  • 競合あり案件比率、失注率、競合金利差、競合集中度を集計する。
  • 時系列データが蓄積した段階で LV 係数推定を検討する。
  • 目的は信用リスクではなく、営業・金利・市場圧力の観測とする。
実装可能性: 高

ステルス競合の逆算

競合名が未入力でも、高スコア失注・失注理由未入力・同業種失注率上昇・金利乖離などから暗黙の競合圧力を推定する。

  • 明示競合と未入力競合を分けてラベル設計する。
  • 新規先、相見積もり、銀行紹介、競合社数を重要特徴として扱う。
  • 効果は本体 AUC よりも、失注予兆と金利提案に出やすい。
実装可能性: 高

メブキちゃん: 現場語への翻訳 UI

AI が発見した市場の歪みを、審査担当者が次に確認できる行動へ翻訳する。判定の断定ではなく、参考所見と確認観点を提示する。

  • 「信用リスク」「競合リスク」「金利リスク」「商流リスク」を切り分ける。
  • 数学用語をそのまま出さず、担当者の確認行動に落とす。
  • 承認・否決の最終判断は人間に残す。

2. 効果可能性

領域 効果可能性 理由 主な評価指標
Q_risk 説明強化 高 既存の量子風解析部品があり、高スコア案件の違和感検知に直結する。 高スコア失注の検出率、二次審査フラグ精度、担当者納得度
ステルス競合推定 高 競合有無、競合金利、競合社数、失注理由、成約/失注の入力導線がある。 後追い確認精度、競合情報入力率、失注理由入力率
新規先の成約予測 中〜高 新規先は商流・競争圧力の影響が大きく、既存特徴量との相性がよい。 新規先 AUC、成約率、金利提案採用率
本体信用スコア AUC 中〜低 過去の QCL 検証では本体モデルへの上乗せ効果が薄かったため、直接混入は慎重にする。 OOF AUC、セグメント別 AUC、キャリブレーション
LV 市場予測 中 概念は有効だが、競合名・失注理由・時系列件数の品質に依存する。 業種別失注率の先行検知、競合集中度、金利乖離
メブキちゃん UI 高 数理モデルを現場語へ翻訳でき、担当者の確認行動を促せる。 確認項目クリック率、担当者フィードバック、説明品質

3. 段階導入ロードマップ

Phase 0: 用語と実務指標の対応付け

AURION の言葉を、審査・営業・モデル評価で扱える指標に翻訳する。

AURION 用語実務上の定義
位相のズレ財務ペア間の干渉乖離、期待レンジからの外れ
特異点高寄与ペア、外挿フラグ、残差シグナルのピーク
市場の歪み業種別失注率、競合集中、金利乖離、スコア乖離
捕食者競合他社、銀行融資、現金購入、設備見合わせ
AI の好奇心モデル監視フックが検出した改善候補

Phase 1: Q_risk の再位置づけ

Q_risk を本体スコアに混ぜるのではなく、二次審査・説明・高スコア違和感検知へ固定する。

  • pair_contributions から上位矛盾ペアを抽出する。
  • ood_flags で過去成約分布から外れた変数を表示する。
  • residual_signal を「説明しきれない違和感」として扱う。

Phase 2: ステルス競合推定

競合名が未入力の高スコア失注や、同業種の失注率上昇を、暗黙の競合圧力として抽出する。

  • 明示競合あり、明示競合なし、競合未入力、ステルス競合候補を分ける。
  • 高スコア失注、失注理由未入力、金利乖離、相見積もりを候補条件にする。
  • 業種・営業部・新規/既存で集計し、週次レビューに載せる。

Phase 3: LV_Environment Lite

厳密な微分方程式より先に、競合圧力指数を時系列で追跡する。

x(t): 自社の成約機会指数
y(t): 競合圧力指数
z(t): 金利防衛余力

x(t+1) = x(t) + 成約増加要因 - 競合圧力影響
y(t+1) = y(t) + 失注集中 + 競合金利差 - 自社防衛成功
z(t+1) = 自社提示余力 - 基準金利 - 競合提示圧力

Phase 4: メブキちゃん UI

数学モデルの出力を、現場担当者が確認できる行動へ変換する。

🌱 メブキちゃんの観測

この案件は信用力そのものより、
「競合圧力」と「設備投資に対する利益の出方」に歪みがあります。

見るべき点:
1. 同業種で直近失注が増えています
2. 競合提示金利との差が大きい可能性があります
3. 営業利益と設備規模の整合に注意してください

次の一手:
- 競合名と提示条件を確認
- 金利ではなく保守・納期・支払条件で差別化
- 高スコアでも二次確認を推奨

Phase 5: Human-approved 自律改善

AI が勝手に本番係数や承認ラインを変えるのではなく、検出、仮説、差分、評価、人間承認の順で進める。

  • 週次レビューで改善候補を提示する。
  • 過去データで AUC、承認率、誤承認、説明品質、セグメント別劣化を確認する。
  • 採否理由と評価結果を保存し、次回改善の学習材料にする。

4. 優先順位

最優先

Q_risk 説明レイヤー化

既存部品があり、実装リスクが低い。高スコア案件の見落とし防止に効く。

最優先

ステルス競合推定

失注・競合・金利情報を活用し、営業打ち手と金利提案に効果が出やすい。

最優先

競合圧力指数

LV 方程式の前段として、業種別・営業部別の歪みを観測する。

中優先

メブキちゃん UI

出力指標が固まった後に、現場語へ翻訳する UI として導入する。

中優先

スマホ向けスライダー

競合圧力、金利弾力性、財務矛盾度など 3〜5 項目に限定する。

後回し

厳密 LV / 本体混入

データ蓄積と検証前に行うと、過学習・説明困難・疑似科学感が強くなる。

5. 推奨アーキテクチャ

AURION CORE は、既存スコアリングを置き換えず、観測・説明・改善提案のレイヤーとして横に置く。

既存スコアリング本体
  ├─ RF/LR 主モデル
  ├─ ベイズ推論
  ├─ 定性評価
  └─ 物件スコア

AURION 観測レイヤー
  ├─ Q_risk: 財務矛盾・位相ズレ検知
  ├─ Stealth Competitor: 暗黙の競合圧力推定
  ├─ LV_Environment Lite: 業種別・営業部別の競合圧力推移
  └─ メブキちゃん: 現場語への翻訳

Human Approval
  ├─ 週次レビュー
  ├─ 過去データ検証
  ├─ 採否理由保存
  └─ 必要なものだけ本番反映
最終判断 AURION CORE は「承認する AI」ではなく、「市場の歪みを見つけ、人間に分かる言葉で知らせる AI」として導入する。これにより、効果・説明責任・運用安全性を両立できる。
