## 全体構造

```
成約可能性スコア
  = 物件スコア × asset_w
  + 成約可能性（借手スコア） × obligor_w
```

---

## 1. 成約可能性スコアとは

**定義**: 「審査が通り、かつ成約できる案件の特徴にどれだけ合致しているか」を0〜100点で表した複合指標。

- 純粋な信用リスク指標ではない
- ロジスティック回帰モデルが成約/失注データで学習しているため、審査通過確率と成約確率が混在している
- この性質を踏まえ「成約可能性スコア」と定義する（旧称: 総合スコア）

---

## 2. 物件スコア（asset_score）

`lease_assets.json` に定義された固定点数。フォームで選択した物件に応じて決まる。

| 物件カテゴリ | 例 |
|------------|-----|
| 車両 | 商用バン: 高め、役員車: 低め |
| 産業機械 | 汎用機: 高め、特注機: 低め |
| 医療機器 | 薬機法リスクに応じて変動 |
| IT機器 | 陳腐化リスクにより全般低め |

---

## 3. ASSET_WEIGHT（物件カテゴリ別配分比率）

物件スコアと借手スコアの配分比率。物件の担保力に応じて設定。

| カテゴリ | 物件ウェイト(asset_w) | 借手ウェイト(obligor_w) | 根拠 |
|---------|:-------------------:|:---------------------:|------|
| 車両 | 35% | 65% | 成熟した中古市場で換金性高い |
| 産業機械 | 25% | 75% | 汎用機は担保価値あるが借手依存大 |
| 医療機器 | 20% | 80% | 薬機法リスクで担保価値が限定 |
| IT機器 | 10% | 90% | 技術陳腐化が速く残存価値ほぼゼロ |

カテゴリ未選択時は回帰最適化重み（`get_score_weights()`）にフォールバック。

## 4. 成約可能性（借手スコア）の計算式

### 4-1. 基準値：3モデル加重平均

```
借手スコア基準値
  = ① 全体モデル × 0.5
  + ② 指標モデル × 0.3
  + ③ 業種別モデル × 0.2
```

各モデルはロジスティック回帰：
```
z = intercept + Σ(変数 × 係数)
スコア = sigmoid(z) × 100 = 1 / (1 + e^(-z)) × 100
```

#### ① 全体モデル（全業種共通）

`customer_type`（既存先 / 新規先）によって使用する係数セットを切り替える。

| 変数 | 単位 | 係数（既存先） | 係数（新規先） | 備考 |
|------|------|:------------:|:------------:|------|
| 切片(intercept) | — | +0.361 | -0.00005 | |
| サービス業ダミー | — | **+1.605** | +0.000121 | 既存先で支配的 |
| 医療・福祉ダミー | — | +0.469 | +0.000017 | |
| 建設業ダミー | — | -0.564 | -0.0000095 | |
| 製造業ダミー | — | -0.251 | -0.0000072 | |
| 運送業ダミー | — | -0.921 | -0.000106 | |
| リース与信残高(log) | 千円→log | **+1.140** | **0（除外）** | 新規先は取引実績なし |
| 既存契約件数 | 件 | +0.281 | **0（除外）** | 同上 |
| 格付4-6 | ダミー | +0.696 | -0.000080 | |
| 格付要注意 | ダミー | +0.222 | +0.000018 | |
| 格付無格付 | ダミー | -0.823 | -0.000016 | |
| 経常利益 | 百万円 | +0.034 | +0.005226 | |
| 家賃費用 | 百万円 | +0.007 | +0.001533 | |
| 売上高(log) | 千円→log | -0.106 | **+0.001121** | 新規先は売上大きい方が有利 |
| 銀行与信残高(log) | 千円→log | -0.045 | +0.001076 | |
| 営業利益 | 百万円 | -0.031 | -0.007541 | |
| 減価償却費 | 百万円 | -0.016 | +0.004581 | |

> 注：係数は「係数分析・更新モード」で成約/失注データを蓄積すると自動更新される
> 注：新規先係数は全般的に小さい（絶対値が小さい）。これはモデルの推定安定性が低く暫定値の側面があるため。

#### ② 指標モデル（財務比率ベース）

業種マッピング：
- D（建設）・P（医療）・H（運送）→ 全体_指標
- I・K・M・R（サービス系） → サービス業_指標
- E（製造） → 製造業_指標

変数：売上高比の各財務比率（営業利益率・粗利率・経常利益率・純利益率・固定資産率・家賃率・減価償却率・機械装置率）

#### ③ 業種別モデル

業種マッピング：
- H → 運送業モデル
- P → 医療モデル
- I・K・M・R → サービス業モデル
- E → 製造業モデル
- D・その他 → 全体モデル

変数：①全体モデルと同じ形式だが業種別に最適化された係数

## 5. 定性補正（任意）

定性スコアリング項目が1件以上入力された場合のみ適用。

```
combined_score = 成約可能性スコア × w_quant + 定性スコア × w_qual
```

- `w_quant` / `w_qual`：デフォルト 60% / 40%（設定変更可能）
- 定性スコア：各項目を5段階評価した加重平均（0〜100点）
- 定性未入力時は `combined_score = 成約可能性スコア` のまま

## 6. 成約可能性スコアとその他指標の関係

```
成約可能性スコア（0〜100点）
        │
        ├──→ 承認判定: スコア ≧ 承認ライン → 承認圏内
        │
        └──→ 軍師モード ベイズ推定承認確率
                prior = スコア/100 × (1 - PD/100)
                posterior = prior
                  + リセール評価（±0.05〜+0.15）
                  + リピート実績（+0.04/件、上限+0.20）
                  + 補助金採択（+0.12）
                  + メイン銀行支援（+0.10）
                  + 業種別成功率（逐次学習）
                display_prob = posterior + フレーズブースト×0.3
                ※BNチェックあり時はBN承認確率でdisplay_probを上書き

BN逆転シミュレーター（独立計算）
  チェックボックス10項目 → pgmpy BayesianNetwork → approval_prob
        │
        ├──→ 軍師 display_prob の上書き（チェックあり時）
        └──→ 借手スコアへの BN補正（係数デフォルト0のため現状ほぼ無効）
```

## 7. 既知の設計上の注意点

| 項目 | 内容 |
|------|------|
| **モデルの学習目的変数** | 成約/失注（審査承認/否決ではない）→ 信用リスクと営業競争力が混在 |
| **②③モデル** | これまで参考表示のみだったが ver.現在から加重平均に統合済み |
| **BN補正係数** | `bn_approval_prob` 等の係数がデフォルト0のためBNシミュレータが借手スコアにほぼ影響しない |
| **純粋な信用リスク分離** | 学習データが成約/失注ベースのため、審査承認/否決データなしでは分離困難 |
