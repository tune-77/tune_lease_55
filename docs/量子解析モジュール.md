# 量子解析モジュール

高スコア（≥80）案件の財務矛盾を量子干渉計算で検出する軽量アドオン。NumPy/SciPy のみ使用。

## ファイル構成

| ファイル | 役割 |
|---------|------|
| `quantum_analysis_module.py` | コアエンジン（QuantumGate クラス） |
| `train_quantum.py` | モデル学習スクリプト |
| `data/quantum_model.joblib` | 学習済みモデル（コミット対象外） |
| `components/industry_quantum_view.py` | Streamlit 表示コンポーネント |

## 入力

`score_calculation.py` から `QuantumGate.predict(case)` が呼ばれる。`case` には以下のキーが必要:

- `op_profit` / `net_income` / `ord_profit` — 利益系（千円）
- `depreciation` — 減価償却費
- `machines` / `equip_total` — 設備系
- `grade` — 格付け文字列（例: `"②B格"`）
- `trend` — トレンド（例: `"1-3"`）
- `industry_sub` — 業種小分類コード（大分類を自動推定）
- `qualit_score` — 定性スコア（任意）

## 出力キー

| キー | 型 | 内容 |
|------|----|------|
| `quantum_risk` | float | 矛盾度スコア（0〜100 程度。高いほど異常） |
| `quantum_verdict` | str | 自然言語での判定文 |
| `pair_anomalies` | list[str] | 矛盾が検出された変数ペアの説明リスト |

`score_calculation.py` は `pair_anomalies` を `quantum_anomalies` として `last_result` にセットする。

## 閾値

| 閾値 | 意味 |
|------|------|
| `quantum_risk < 20` | 正常（レポートで緑表示） |
| `20 ≤ quantum_risk < 35` | 軽微な矛盾（黄表示） |
| `quantum_risk ≥ 35` | 要注意 — `needs_secondary_review` フラグが立つ（赤表示） |

35 という閾値は、過去案件で二次審査に回った案件の quantum_risk 分布の中央値付近から設定。

## 業種別ペア重み

業種大分類ごとに追加の矛盾ペアと重みが定義されている:

| 業種 | 追加ペア例 |
|------|-----------|
| D 建設業 | 利益 vs 機械装置（重み 2.0）、利益 vs 設備合計（2.5） |
| H 運輸業 | 機械装置 vs 利益（2.2）、減価償却 vs 機械装置（2.0） |
| P 医療・福祉 | 純利益 vs 営業利益（2.5）、経常利益 vs 営業利益（2.0） |

業種が不明な場合は基本ペア（利益 vs 減価償却 等）のみ使用。

## レポート・CSV への出力

- **レポート画面**: 「⚛️ 量子整合性チェック」セクションに quantum_risk・verdict・上位3異常ペアを表示
- **CSV ダウンロード**: `quantum_risk` / `quantum_verdict` カラムに出力

## 再学習

```bash
python train_quantum.py
# → data/quantum_model.joblib を上書き
```

学習データは SQLite の過去案件（`data/lease_data.db`）から自動取得する。
