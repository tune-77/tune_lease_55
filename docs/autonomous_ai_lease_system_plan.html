01 / GOAL

目指す姿

審査AI → 改善する審査OS

判断品質を上げる

スコア・金利・コメントの精度を案件実績で継続改善し、失注・貸倒・例外承認の原因を自動で説明する。

改善速度を上げる

人間が「改善テーマ」を探す前に、AIがログを読み、仮説、影響範囲、実装差分、テストまで用意する。

統制を強める

勝手に本番反映せず、モデル変更・ルール変更・説明文変更を記録し、承認者がレビューできる形にする。

重要方針: 自律化とは「AIが勝手に審査判断を変える」ことではありません。AIが改善候補を発見し、根拠と検証結果を添えて提案し、人間が承認して反映する仕組みにします。

05 / GOVERNANCE

絶対に入れるべき安全装置

金融業務向け

Human-in-the-loop

  • スコア係数、承認ライン、金利ロジックはAIが直接本番変更しない。
  • 変更はPR、評価レポート、承認ログを必須にする。
  • AIコメントは「参考所見」であり、最終判断者を画面に明記する。

評価とロールバック

  • 本番前に過去案件で回帰テスト、セグメント別AUC、キャリブレーションを確認。
  • モデル・ルール・プロンプトはバージョン管理し、いつでも戻せるようにする。
  • AI提案の採否理由を保存し、次回改善の学習材料にする。

禁止する自律化

  • AIが単独で承認/否決を確定する。
  • AIが監査ログなしで係数・閾値を更新する。
  • 根拠のない外部情報で審査結果を変える。

許可する自律化

  • 改善候補の発見、仮説作成、実験ブランチ作成。
  • テスト実行、評価レポート作成、PR作成。
  • 審査担当者への質問案、追加資料依頼案の生成。

07 / STARTER PROMPTS

すぐ使える実装プロンプト例

コピペ用

Claude Plannerに渡す

あなたはリース審査AIの改善責任者です。 過去30日分の案件ログ、失注理由、スコア分布、AIコメント修正履歴を読み、 1. 精度改善 2. 業務効率化 3. 監査・説明責任 の観点で改善候補を最大5件出してください。 各候補には、期待効果、リスク、必要データ、Codexに依頼する実装タスクを付けてください。

Codex Builderに渡す

リース審査AIの改善候補「{テーマ名}」を実装してください。 要件: - 既存スコアリング結果を壊さない - 単体テストまたは回帰テストを追加 - 変更理由と評価方法をdocsに追記 - モデル・係数・プロンプトを変更する場合はバージョンとロールバック方法を記録 - PR本文に検証結果をまとめる

Gemini APIに渡す

あなたはリース審査担当者を補助するAIです。 以下の案件について、スコアと財務指標に矛盾しない審査所見を作成してください。 必ず以下を分けてください: - 承認材料 - 懸念材料 - 追加確認したい資料 - 金利・条件交渉の論点 断定しすぎず、最終判断は人間が行う前提で書いてください。

Evaluatorに渡す

変更ブランチと現行本番を比較してください。 評価軸: - 全体AUC / セグメント別AUC - 誤承認・誤否決の増減 - 承認率と収益性への影響 - AIコメントのルーブリックスコア - 既存テストの失敗有無 結論は「採用」「条件付き採用」「不採用」で出してください。
