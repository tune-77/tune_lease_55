## 必須技術チェック（このリポジトリでの充足状況）

- [x] Google Cloud アプリケーション実行プロダクト: **Cloud Run**（API/Web分離デプロイ）
- [x] Google Cloud AI 技術: **Gemini API**（OCR / ストリーミング / 複数紫苑討論 / Research）+ **ADK**（`api/shion_agent.py`）
- [ ] （任意）Speech-to-Text / Text-to-Speech: 現状 `/voice-chat` はブラウザ Web Speech API。Google Cloud 公式APIではないため任意加点にはならない。工数対効果が低いため今回は見送り、余力があれば次点候補。

## リポジトリ側で対応済み

## ここから先は人（提出者）が行う必要がある作業

以下はこの実行環境（サンドボックス、GCP認証情報なし）からは実施できません。提出者本人の環境で対応してください。

1. **ライブデモURLの確定**: `PUBLIC_TUNNEL=1` の quick tunnel ではなく、Cloud Run の安定した本番URL（`tune-lease-55-web` / `tune-lease-55-api`）を最終確認する。決勝が8/19まであるため、審査期間中ずっと生きているURLが必要。
2. **デプロイ状態の再確認**: `python3 scripts/check_cloudrun_demo_readiness.py --base-url https://YOUR-CLOUDRUN-URL` を実行し、`CLOUDRUN_DATA_MODE=demo` で動いていることを確認する。
3. **デモ動画の収録**（3分目安の構成案）:
   - 0:00-0:20 課題提示（「プロトタイプは作れるが実運用まで持っていけない」への回答であることを一言で）
   - 0:20-0:50 必須技術（Cloud Run + Gemini + ADK）を画面かスライドで明示
   - 0:50-1:40 デモ本編: サンプル案件読み込み → 審査実行 → 軍師AI（ADKエージェント）のツール呼び出しがストリーミング表示される様子 → 紫苑レビュー
   - 1:40-2:20 DevOpsサイクル図（README追加分）を見せながら「demo/本番分離 → 検疫 → 人間承認 → 昇格」を説明
   - 2:20-3:00 まとめ（個人開発1名であることも含め、実運用設計の作り込みをアピール）
4. **提出フォーム記入**: 特設サイトの提出要項（提出物形式・GitHub公開範囲・チーム情報欄など）を実際のページで再確認し、リポジトリURL・動画URL・概要文を記入する。
5. **1人チームである旨の確認**: 参加要項上、個人でも参加可能だが、提出フォームのチーム欄・決勇進出時の対応可否（8/19オフライン参加が1人で可能か）を事前に確認しておく。
