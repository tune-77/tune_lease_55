# 2026-07-16

## Hackathon / Shion Judgment Asset Framing

- User clarified that Shion's purpose should not be framed as simply increasing close rate or user satisfaction. The safer and stronger goal is improving the reproducibility and transferability of lease judgment: preserving why a human noticed something, corrected it, added a condition, or changed the review point.
- Judgment assets do not need to prove universal correctness at creation time. They should first prove behavior change: the same case reviewed before and after accumulated judgment assets should move from generic checks to more concrete review conditions, counterpoints, and ringi wording.
- The "1000-case fast-forward demo" must not imply that 1000 real cases were processed unless the backing data exists. If used before real data exists, call it a simulation. If the final stage is reached, create real dummy-case judgment logs and JSON assets so the demo has artifacts behind it.
- More useful than weak volume: create around 30 strong common judgment assets first, then 20-30 per major industry. Each asset should be about 120-250 Japanese characters and follow: `条件 -> 見る論点 -> 稟議に残す一文 -> 使わない条件`.
- Key concept saved: human comments must become program-like. Shion should convert vague human discomfort into reusable judgment code / prompt assets. This is not casual prompt engineering; it is an engineering loop for business judgment prompts.
- Presentation framing: Shion is a judgment-prompt PDCA platform. It structures human judgment, saves it as reusable prompt assets, recombines it for later cases, receives human feedback, and improves the judgment asset set.
- User decided the next demo-data concept is "judgment abstraction": like mathematics abstracts concrete things into numbers/formulas, Shion abstracts individual case hesitation into judgment hypotheses, conditions, counterpoints, outcomes, verification, and judgment assets. Demo data should include dummy outcomes such as contracted/lost/rejected/delinquency, but the relationship between judgment hypothesis and result reason must be coherent.
- User decided to stop treating Private Reflection as "pretending to introspect." The new rule is: the source of truth is a judgment change log, not literary self-talk. Shion should first structure previous input, previous judgment, human correction, missed point, next checks, judgment asset candidate, and remaining uncertainty. A novelist layer may then translate that log into narrative.
- User chose ツンコ and ユウケイ as the protagonists for the novelist layer. They are expression-layer characters, not the source of truth. The truth remains the judgment change log.
