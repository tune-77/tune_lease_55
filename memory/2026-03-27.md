## Events
- Terminal.app crashed with `EXC_CRASH (SIGABRT)` due to memory corruption in `nanov2_calloc_type`.
- Analysis showed the crash occurred during `NSPersistentUIManager flushAllChanges`, which is part of macOS's state restoration mechanism.
- Reason: Corrupted saved state or potentially excessive data being saved during background flush.

## Actions Taken
- Analyzed the crash log provided by the user.
- Suggested clearing the saved state directory: `rm -rf ~/Library/Saved\ Application\ State/com.apple.Terminal.savedState`.
- Recommended checking "System Settings > Desktop & Dock > Close windows when quitting an app" to prevent persistence issues.
