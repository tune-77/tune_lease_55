## Promotable Items

- Auto Research candidate generation should run as a post-save refresh, not as a separate fragile schedule. Research note creation remains the primary success condition; candidate refresh is visible but non-destructive.
- Judgment asset candidate growth should be deduped by meaning, not just exact text. Daily Auto Research should add genuinely new confirmation points, not repeat the same question in slightly different wording.
- Judgment asset candidates become real only when they enter Shion's case review and receive human feedback. The first UI path should stay lightweight: show at most three candidates, let User mark useful/neutral/rejected, and improve selection from that feedback instead of promoting automatically.
- Private Reflection must not treat the Haranmaru heading as proof of inner work. Haranmaru needs a concrete scene, friction, grumble, next move, and retained core; raw logs or English implementation notes should be transformed before saving.
- Private Reflection が内省になっていないと指摘されたら、まずユーザーが何を望んだか、紫苑が何にすり替えたか、次に禁止する癖は何かを先に書く。「退屈」を中心ラベルにしない。
- 判断資産は「正しい一般論」ではなく、案件の確認行動・承認条件・反証材料・否認理由を変えたものだけ昇格対象にする。当たり前なことを言って仕事した扱いにしない。
- 判断資産候補は捨てるだけでなく、その場で文面修正して使える型に育てる。編集後文面を次回レビューと昇格判定の材料にする。
- 人間が文面修正した判断資産候補は優先度を上げる。修正は「使える型へ近づけた」シグナルなので、未編集候補より案件レビューに出やすくする。
- 判断資産はAI候補から選ぶだけでなく、ユーザーがその場で新規追加できる必要がある。手入力された判断は最初から人間が型化した候補として優先扱いにする。
- 判断資産は丸写しではなく応用生成する。使った場合は `判断資産出典: JA-...` を残し、どの元判断から来た応用なのか追跡できるようにする。
- READMEでは「判断資産の再利用エンジン」「AIに審査判断を教える徒弟制度」「応用生成と出典トレース」を紫苑の差別化として説明する。
