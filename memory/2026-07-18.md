# 2026-07-18

## 紫苑: Obsidian / Cloud Run / セキュリティ説明の修正

- 事実: 現在の Obsidian Vault は Mac ローカルアプリで開いているが、実体は iCloud Drive 同期領域にある。
- 影響: 「ローカル保存だから安全」と言い切る説明は不正確。発表・README・SystemOverview ではこの表現を避ける。
- 次の行動: セキュリティ説明は「AI/Cloud Run が判断資産の正本を直接書き換えない」「審査DBと判断資産を分離」「改善候補は検疫・承認後に昇格」「正本の保存先は運用ポリシーに応じて差し替え可能」に寄せる。

推奨文言:

```text
現在のプロトタイプでは、判断資産の正本は iCloud 同期された Obsidian Vault に置いています。
Cloud Run は正本を直接書き換えず、同期コピーと記憶インデックスを読んで判断に使います。
本番化する場合は、組織のセキュリティ要件に応じて、ローカル専用 Vault、社内管理ストレージ、GCS、Git 管理 Vault などへ置き換えられます。
重要なのは保存先そのものではなく、AI が正本を勝手に更新せず、検疫・人間承認・昇格を通すことです。
```
