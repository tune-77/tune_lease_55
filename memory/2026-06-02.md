## Codex 自動改善キュー

- 改善リストから Codex が安全に触れそうな候補を毎朝最大3件に絞る `scripts/build_codex_auto_queue.py` を追加。
- 4:00 の `scripts/run_daily_improvement_pipeline.sh` でキューを生成し、`reports/latest.json` へ件数メタデータを同期する方針にした。
- 6:00 の `scripts/aurion_core_daily.py --mode morning-report` は `Codex Auto Improvement Queue` として安全候補数、今日のキュー、保留候補、手動/ブロック件数を朝レポートと通知に出す。
- 現時点では「自動でコード修正まで実行」ではなく、朝の報告に実行候補を出す段階。実行まで自動化する場合は Codex の安定した実行エントリポイントと失敗時停止条件を別途固定する。

## 改善申告漏れチェック

- 手動またはCodexで直したのに `REV-xxx` を済登録し忘れるケースに備え、朝レポートへ `Improvement Declaration Gap Check` を追加。
- 直近36時間のgitコミットと最新改善レポートを突合し、明示REV-IDが未登録のまま残っているもの、またはREVなしコミットとタイトルが近いものを「申告漏れ疑い」として表示する。
- 自動で `applied` にはしない。誤って済にするより、疑いとして朝報告へ出して人間確認に倒す方針。
