## Future countermeasures
- When user says "home won't open", do not stop at `curl 200`. Check whether the page is returning a loading-only shell or the actual home content.
- Prefer this triage order: `curl -i /home`, Next log, API log for `/api/dashboard/stats`, current Cloudflare tunnel log URL, then frontend code for full-screen loading/error gates.
- For Cloudflare quick tunnels, always refresh the URL from the newest `logs/next/tunnel_*.log`; old trycloudflare URLs are disposable.
- If local sandbox `curl` cannot reach `127.0.0.1:3000` while `next-server` is alive, retry with approved/escalated `curl --max-time` before assuming Next is down.
