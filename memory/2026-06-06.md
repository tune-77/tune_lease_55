- User reported that the home page would not open. HTTP checks showed `/` and `/home` returned `200 OK`; API logs also showed `/api/dashboard/stats`, `/api/lease-news/recent`, and related endpoints returning `200`.
- Root cause: `frontend/src/app/home/page.tsx` rendered a full-screen `loading` state (`データを集計中...`) until dashboard stats finished. This made the home page appear unavailable even though server/API routing was healthy. Cloudflare URL changes and browser/service-worker cache made diagnosis slower.
- Fix applied: removed the full-screen early `if (loading) return ...` gate and changed the status badge to `ONLINE & SYNCING` while data is still loading, so the home dashboard shell opens immediately.

## Future countermeasures
- When user says "home won't open", do not stop at `curl 200`. Check whether the page is returning a loading-only shell or the actual home content.
- Prefer this triage order: `curl -i /home`, Next log, API log for `/api/dashboard/stats`, current Cloudflare tunnel log URL, then frontend code for full-screen loading/error gates.
- For Cloudflare quick tunnels, always refresh the URL from the newest `logs/next/tunnel_*.log`; old trycloudflare URLs are disposable.
- If local sandbox `curl` cannot reach `127.0.0.1:3000` while `next-server` is alive, retry with approved/escalated `curl --max-time` before assuming Next is down.
