## Decisions
- 軍師の SSE 経路も通常チャットと同じく、PDCA 注入と差分ログを残す方針にした。
- 修正の登録はモーダルなしの1クリック追記にし、判断負荷を増やさない。
- 月次レポートは既存の自動集計と Obsidian 連携を維持し、パイプラインの拡張は最小限に留める。
- 再帰的自己改善は `reports/recursive_self_improvement_latest.json` と `reports/latest.json` の両方で観測する。
- 再帰的自己改善は観測はするが、既定では `reports/latest.json` を書き換えない。
- 仕様は `specs/phase6/P6-001-recursive-self-improvement.md`、正本一覧は `docs/improvement_source_of_truth.md` で確認する。
- いまの主眼は審査機能の拡張より、AI チャットの知識を深めることに置く。
- 知識進化の評価は `知識化率` `再利用率` `効果率` `重複・陳腐化率` の4指標で見る。
- 改善ログの1行テンプレートは `【知識KPI】{指標}: {現状} -> {目標} / 変化: {何が変わるか} / 根拠: {観測できる証拠} / 次回: {いつどう確認するか}` とする。
- 知識KPIテンプレートは内部運用用に保持し、画面には出さない。
- 固定ルールは増やさず、今ある仕組みを止めずに回す。
- リース知性体として重要なのは、仕組みを増やすことではなく、止めずに回し続けること。
- 世界初のリース知性体を目指す。

## Promotable
- Prompt feedback は `PDCAあり/なし` と `前回応答との差分` を同じ基盤で記録すると、Next 側でも効果検証しやすい。
- UI 側の集計カードは、改善ループが動いているかを毎回手で掘らずに確認できる入口になる。
- 修正を教える最短経路は、改善ログの「修正登録」ボタンから `ai_prompt_addons` を増やすこと。
- 直近の重点は審査ロジックより、AI チャットの知識・応答品質を厚くすること。
- 月次レビューでは、作業件数よりも「知識が実際に再利用されたか」を優先して見る。
- 曖昧な要約より、測定可能な変化を書く。
- 月次レビューでテンプレートが実際に使われているかだけ確認する。
- 迷ったら追加実装ではなく、現行の知識ループが回っているかを見る。
- 追加より継続を優先する。
