# 件数要件一覧（lease_logic_sumaho10）

| 機能 | 必要な件数 | 備考 |
|------|------------|------|
| **定性要因分析**（LR・LightGBM・アンサンブル） | **成約+失注 50件以上** | それ未満は「50件以上で利用できます」と表示され実行不可。 |
| **定量要因分析（全体）** | **成約+失注 50件以上** | 同上。 |
| **定量・業種ごと** | 全登録 **10件以上** で実行可能。業種ごとは **5件以上** あれば分析し、**50件未満なら50件にブートストラップ**して学習。 | 5件未満の業種は「データなしまたは5件未満」でスキップ。 |
| **定量・指標ごと** | 全登録 **10件以上** で実行可能。指標ごとは **5件以上** あれば分析し、**50件未満なら50件にブートストラップ**。 | 同上。 |
| **成約の正体レポート** | **成約 5件以上** | 成約だけの件数。5件未満は「成約データが5件以上貯まると表示されます」と表示。 |
| **係数分析・更新（β）** | **成約+失注 5件以上**（業種/指標モデルごとに5件以上ある組だけ回帰） | 5件未満の組は「データ不足」としてスキップ。 |

---

## まとめ

- **定性・定量の「全体」分析を使いたい**: **成約+失注で 50件** を目安に貯める。
- **業種ごと・指標ごと**まで使う: 同じく全体で **50件** が望ましいが、**10件以上**あれば実行可能。業種/指標ごとに **5件以上** あればその組は分析され、**50件未満の組は自動で50件にリサンプル**して学習する。
- **成約の正体レポート**だけ見たい: **成約 5件** で表示可能。
- **係数更新**だけしたい: **成約+失注 5件** で実行可能（業種・指標ごとはそれぞれ5件以上ある組のみ更新）。

定数 `QUALITATIVE_ANALYSIS_MIN_CASES = 50` は `lease_logic_sumaho10.py` 内で共通参照されているので、ここを変更すると「50件」がかかわる分析の閾値が一括で変わります。
