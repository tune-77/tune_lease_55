# 簡易スコア（項目絞り）と全項目入力のスコア差

## 結論

- **典型的な案件**: 省略項目を 0 にしても **スコア差はおおむね 2〜4 点程度**。
- **減価償却・賃借料が大きい案件**: 最大で **5〜10 点程度** ずれることがある。
- 本審査で使っている**訂正（補正）**も簡易スコアでは**一切行っていない**（下記）。

---

## 簡易スコアで抜けている「訂正」部分

本審査（Streamlit）では、借手スコアに**訂正**を加えた「契約期待度」を別指標として計算・表示している。簡易スコアではこの訂正を**すべて行っていない**。

| 訂正項目 | 内容 | 簡易スコア |
|----------|------|------------|
| メイン先 | 取引区分（メイン先で +5 点程度） | 未実装 |
| 競合 | 競合あり/なし（競合なしでプラスなど） | 未実装 |
| 業界景気 | ネット検索の業界リスクを Z 化して加減 | 未実装 |
| 定性スコア | 強みタグ・熱意テキストの加点 | 未実装 |
| 自己資本比率の追加効果 | 業界目安とは別の「1%あたり」の効果 | 未実装 |
| 金利差（競合比） | 自社利回りと競合提示金利の差 | 未実装 |

- **注意**: 本審査の**承認/要審議の判定**は「総合スコア」（借手スコア×重み＋物件スコア×重み）で行っており、上記の訂正は**契約期待度**にのみ反映され、判定スコアには直接足し込んでいない。簡易スコアは「総合スコア」に相当する式だけを実装しているため、訂正をしていない分、**契約期待度**の値とは一致しない（契約期待度そのものは簡易版では出していない）。

---

## 理由（係数ベースの試算）

簡易スコアで「未入力＝0」になる項目と、係数（全体_既存先）は次のとおり。

| 項目 | 係数 | 100百万円(10億円)あたりの z への効き |
|------|------|--------------------------------------|
| gross_profit（粗利） | 0.00021 | +0.02 程度 |
| machines（機械装置） | 0.00028 | +0.03 程度 |
| other_assets（その他固定資産） | -0.00117 | -0.12 程度 |
| rent（賃借料・資産） | 0.0039 | +0.04 程度 |
| depreciation（減価償却） | -0.00289 | -0.03 程度 |
| dep_expense（減価償却費） | **-0.016** | **-0.16 程度** |
| rent_expense（賃借料・経費） | 0.007 | +0.07 程度 |

- スコアは **借手スコア = sigmoid(z)×100** で、z が 0.1 変わるとおおよそ **1〜3 点** 程度変動する（z の水準による）。
- 上記の「省略項目」をすべて 0 にしたとき、**通常の規模**（粗利・機械・減価償却・賃借料などが数億〜数十億円）だと、z の合計への影響は **おおよそ ±0.05〜0.15** 程度になることが多い → **スコア差 2〜4 点程度**。
- **減価償却費・賃借料が大きい**案件では、dep_expense と rent_expense の効きが効いて、z が 0.2〜0.4 程度動くこともある → **スコア差 5〜10 点程度**になりうる。

---

## 使い分けの目安

- **簡易スコア（Flask）**: おおよその水準・承認ライン感を見たいとき。財務の省略項目（0 扱い）による 2〜4 点差と、訂正（メイン先・競合・定性等）をしていないことは許容する。
- **本審査（Streamlit）**: 全項目入力＋訂正＋学習モデル否決ペナルティ・利回り等を含め、正式な判定と契約期待度に使う。
