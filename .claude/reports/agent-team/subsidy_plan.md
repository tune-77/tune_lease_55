# リース補助金活用 PLAN

## 概要
審査AIシステムに補助金マスタ（SQLite管理）と自動マッチングロジックを追加し、
審査レポート画面で「補助金込みシミュレーション」を可視化することで、
顧客の意思決定支援と営業の商談ツール強化を同時に実現する。

## 実施項目

| 優先度 | 施策 | 担当機能 | 工数感 |
|--------|------|---------|--------|
| 🔴 高 | 補助金マスタテーブル設計（業種×設備×地域×確度） | scoring/ + SQLite | 2日 |
| 🔴 高 | 審査レポートに「補助金込み月額シミュレーション」パネル追加 | components/report.py | 3日 |
| 🔴 高 | 補助金マッチングロジック（業種×設備で自動抽出） | scoring/subsidy.py（新規） | 3日 |
| 🟡 中 | 補助金加点をスコアリングに組み込み（確度×軽減率） | scoring/score_engine.py | 2日 |
| 🟡 中 | 補助金マスタ管理UI（Streamlit管理ページ） | components/admin_subsidy.py（新規） | 4日 |
| 🟡 中 | 補助金カード表示（フォーム完了後3件表示、F字動線） | components/chat_wizard.py | 2日 |
| 🟢 低 | 補助金込みシミュレーションPDF出力 | components/report.py | 5日 |
| 🟢 低 | 外部補助金API連携（J-Net21等） | 新規モジュール | 2〜3週間 |

## 実装ロードマップ
- Phase 1（即時〜3日）: 補助金マスタテーブル設計 + マッチングロジック基礎実装。scoring/subsidy.pyを新規作成し、業種×設備×地域の3軸マッチングを実装。
- Phase 2（1〜2週間）: レポート画面へのシミュレーションパネル追加 + スコアリングへの補助金加点組み込み + 補助金カード表示（フォーム完了後）。
- Phase 3（1ヶ月〜）: 管理UI整備 + PDF出力対応 + 将来的な外部API連携の基盤づくり。

## リスク・課題
- 補助金情報は制度変更が頻繁なため、マスタの鮮度維持が継続コスト（運用担当の確保が必要）
- 補助金「取得確度」の設定基準が曖昧だとスコアへの加点が恣意的になる→確度定義を明文化すること
- 外部APIは仕様変更リスクが高く、スクレイピングは利用規約違反の可能性があるため手動更新方式を基本とする
- PDF出力ライブラリ（reportlab等）の追加依存が生じる
